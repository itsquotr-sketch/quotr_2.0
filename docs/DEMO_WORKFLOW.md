# Quotr demo workflow

Manual QA flow for test users and internal verification. Do not seed production automatically.

## Sample project brief

Use this brief when creating a test project:

> **Title:** Deck and outdoor — Smith residence  
> **Client:** John Smith  
> **Site:** 12 Example Road, Auckland  
> **Brief:** Client wants a 5m x 11m hardwood deck with stairs and balustrade, removal of existing deck, plus a 13m² pergola and 15lm fence replacement. Standard site access. Client wants work completed within 6 weeks of acceptance.

## Expected work areas

After analysis and confirmation, expect work areas similar to:

- Demolition / removal
- Substructure / framing
- Decking
- Stairs and balustrade
- Pergola
- Fencing
- Finishing

Exact names depend on AI output and your org work-area catalogue.

## Workflow steps

1. **Dashboard** — Create the project from the sample brief.
2. **Project Assistant** — Add site notes if useful. Run **Analyse job**.
3. **Work areas** — Confirm or adjust suggested work areas.
4. **Questions** — Answer scope questions where prompted.
5. **Scope review** — Review scope summary before estimating.
6. **Quick estimate** — Generate estimate. Review totals and line items.
7. **Final pricing** — Prepare final pricing from the estimate.
8. **Edit pricing** — Change one line item label or sell price; save.
9. **Review** — Complete the review checklist; mark pricing as reviewed.
10. **Quote** — Create quote from reviewed pricing.
11. **Quote page** — Review client-facing preview; edit scope/terms if needed.
12. **Print** — Use **Print / Save as PDF**; disable browser headers/footers in the print dialog.
13. **Revision** — Optionally create a revision after a pricing change.
14. **Dashboard** — Archive or duplicate the project; confirm dashboard filters.

## Print output checks

- Company name, contact, GST/NZBN visible
- No internal cost, margin, productivity or rate-source wording in line items
- Assumptions/exclusions/terms on a separate final page
- Totals: subtotal, GST, total incl. GST

## Recalibration test

1. Regenerate the estimate after editing scope.
2. Open Final Pricing — recalibration banner should appear.
3. Preview recalibration — if no changes, message should explain pricing already matches estimate.
4. Apply or keep current pricing as appropriate.

## Feedback

Use **Report issue** in the account menu or sidebar. Include the page URL in the email body.
